# 本地部署开发版本【支持代码热更新，面向开发者】

## Linux/MacOS

1. Clone 当前仓库的 dev 分支【最新版本】

```sh
git clone --branch dev --depth 1 git@github.com:ScienceOL/PROTIUM.git
```

2. 进入文件夹并创建环境变量文件

```sh
cd PROTIUM
cp ./docker/.env.example ./docker/.env.dev
```

3. 一键启动项目

```sh
./launch/dev.sh
```

<Note>
使用 -d 参数以后台运行方式，更多参数可使用 -h 查看
</Note>

4. 开发

项目启动后，你可以访问：http://127.0.0.1:32234 进入项目。

现在，你可以修改代码，反馈将立即呈现。更多开发帮助请参阅[开发者指南](/development/quick-start)

<Note>
初次启动项目时，将创建管理员账户，默认凭据为：

* 账号：`admin@example.com`
* 密码：`admin`

你可以通过 .env.dev 文件中的 Init Configuration 部分修改默认创建的管理员账户信息
</Note>

5. 可选配置项
  * 配置 VSCode 工作区

  如果你使用 vscode 开发，可配置 vscode 工作区以减少代码风格上的问题检出

  ```sh
  cp ./.vscode/settings.example.json ./.vscode/settings.json
  ```
